exhaustMap
A pipeable, transformation operator
What It Does:
  • As its source Observable emits, exhaustMap subscribes to and returns the value of its inner Observable and will ignore all emissions from its source Observable until the inner Observable completes. When the inner Observable completes, exhaustMap will once again start listening to emissions from the source Observable.
When To Use It:
  • Use exhaustMap on any backend call that is generated by a UI control that can make multiple calls if concatMap or mergeMap is used instead. For example, 'Refresh' button.
How It Differs From the Other Higher Order Mapping Operators
  • Unnecessary backend calls made by the user are ignored.
