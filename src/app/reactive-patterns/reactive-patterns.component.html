Reactive Programming
A programming paradigm which involves data streams (i.e. Observables) and achieves results via change propagation.
When To Use It:
  • For more complex user interfaces that consist of data retrieved by a multiple API calls
Patterns We'll Use in Our Reactive Programming
Observable Data Services
Data services which act as the sole sources of data retrieval and modification via Observables and public methods.
HttpClient Service
Use to make all CRUD (create, read, update, delete) back end calls. All of its methods return an Observable.
State Management
Use RxJS Subjects to manage the state of our application.
Subjects
We'll create subjects that all have privately exposed observer methods and publicly exposed Observable methods.
Async Pipe
Use the Async pipe in Angular templates to automatically subscribe and unsubscribe to observables.
RxJS Operators
  • We will use RxJS Creation Operators to create our own observable streams.
  • We will use RxJS Pipeable Operators to transform and manipulate observable streams.
